# 文案模板

## 核心原则

### 风格要求
- **语气**：专业实用但略带网感，"朋友式分享"
- **表达**：敢于输出主观观点，口语化，有趣、有价值、有深度
- **去AI模板化**：禁止说明书式内容

### 语言风格
- 全程大白话，专业词立刻通俗化
- 加入博主口头禅、吐槽、主观观点
- 多站在用户视角思考，共情用户痛点

### 互动引导
- 结尾设置具体场景化问题或选择题
- 钩子（如"关注我，每日带你了解一个..."）

---

## 标题公式

### A. 痛点吐槽型（推荐）
```
{痛点吐槽}？{简单解决方案}，{惊喜结果}
```
> 例：AI写的代码不敢用？这个工具帮你盯着，太香了

### B. 反差型
```
{预期外的事实}，{主观评价}
```
> 例：GitHub 25k star 的监控工具，竟然开源免费？

### C. 场景型
```
{场景}+{情绪词}，{解决方案}
```
> 例：产品上线翻车？用这个提前发现问题

## 禁止出现

❌ 说明书式
- "支持XX功能"
- "提供XX能力"

❌ 空洞口号
- "让XX更高效"
- "提升XX体验"

❌ 没有感情的陈述句
- "XX是一个XX平台" → 改成"说实话，XX真的..."

❌ 没有互动的结尾
- 直接结束 → 加上"评论区告诉我"或"关注我"

❌ **同一素材重复使用（强制禁止）**
- 每个场景必须使用不同的图片/截图素材
- 禁止两张及以上场景使用相同素材（包括 banner、仓库卡片、界面截图等）
- 若素材不足，应：①下载更多真实截图，②裁剪素材不同区域，③用不同动画效果区分
- **违规示例**：real_github_card.png 同时用于场景1、5、7 → 必须替换为3张不同素材
- **正确做法**：8个场景 → 至少5-8张不同素材，每张素材最多使用1-2次且需配合不同动画效果

## 素材规则（强制）

### 核心原则：每个场景必须有独特素材
- **严格禁止重复**：8个场景必须使用至少 5-8 张不同素材
- 真实截图优先：优先使用 GitHub README 图片、产品官网截图、UI 截图
- 程序化素材兜底：只有真实素材不足时才用 Canvas 生成，且生成时需传入正确的产品参数

### 素材规划流程
1. 列出该产品已有的真实截图（官方图、GitHub 图片、产品界面）
2. 统计场景数量，确保素材数 ≥ 场景数
3. 若素材不足：
   - 下载 GitHub 仓库的 README 图片（raw.githubusercontent.com）
   - 访问产品官网截图
   - 裁剪同一张图片的不同区域（上下/左右不同裁切）
4. 规划素材分配：确保每张素材最多使用 1-2 次
5. 搭配不同 Ken Burns 动画效果（同一素材第二次使用时换不同效果）

### 真实截图下载方法
```javascript
// GitHub README/Banner 图片
const url = `https://raw.githubusercontent.com/{owner}/{repo}/main/{path}`;
// FFmpeg 转换 webp → png
ffmpeg -i input.webp output.png
```

### 素材命名规范
```
{产品名}_{类型}_{序号}.{ext}
// 例如：trendradar_banner_01.png, langfuse_ui_01.png, deerflow_readme_01.png
```
